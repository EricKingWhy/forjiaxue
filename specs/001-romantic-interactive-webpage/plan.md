# Implementation Plan: ForJiaXue - Romantic Interactive Webpage

**Branch**: `001-romantic-interactive-webpage` | **Date**: 2026-07-06 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/001-romantic-interactive-webpage/spec.md`

## Summary

Build a romantic interactive webpage experience for a loved one. Features include: entry screen with petal animation and optional password, photo particle aggregation with 60k/30k/12k tier system, 3D cylindrical photo wall with music-driven effects, hand gesture unlock (MediaPipe), and finale with typewriter blessing text. Backend provides photo/music upload, EXIF removal, WebP conversion, particle map generation, visitor statistics (hashed IPs), and secret message feature. Mobile-first vertical scroll navigation with performance auto-detection.

## Technical Context

**Language/Version**: TypeScript 5.x (frontend), Python 3.11 (backend)

**Primary Dependencies**: 
- Frontend: Next.js 14+, React 18+, React Three Fiber, Three.js, GSAP, Framer Motion, Zustand, Web Audio API, MediaPipe Hand Landmarker
- Backend: FastAPI, SQLite, SQLAlchemy 2.0+, Pillow, OpenCV

**Storage**: SQLite at `backend/data/app.db`, uploads directory structure: `original/`, `webp/`, `thumb/`, `music/`, `particle_map/`

**Testing**: Jest (frontend unit), Playwright (E2E), pytest (backend)

**Target Platform**: Modern browsers (WebGL, Web Audio API support), mobile-first (iPhone 16 Pro Max as high-end reference), auto-adjusts for low-performance devices

**Project Type**: Web application (frontend + backend)

**Performance Goals**: 
- 60fps particle animation on high/medium tier
- 45fps minimum on low tier
- Photo aggregation in 5-8 seconds
- Complete experience in <3 minutes
- Music continuous playback

**Constraints**: 
- Photo: jpg/jpeg/png/webp, max 10MB
- Music: mp3/m4a/wav, max 20MB
- EXIF must be removed
- IP hashed only (no raw IP storage)
- Bloom auto-disabled on low-performance

**Scale/Scope**: 
- 1 main photo + 6-12 wall photos
- 1 active music track
- Single admin user
- Personal use (low traffic, ~100 visits/month estimate)
